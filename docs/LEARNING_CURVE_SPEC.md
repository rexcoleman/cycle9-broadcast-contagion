## 1) Training Fractions

**Fractions:** {{TRAINING_FRACTIONS}} (default: `[0.1, 0.25, 0.5, 0.75, 1.0]`)

- Minimum **5** distinct fractions required.
- Fractions are applied to the training set only. The validation and test sets remain unchanged.
- Sampling is stratified to preserve class distribution at each fraction.
- For each fraction, the same random seed controls which samples are selected, ensuring reproducibility.

---

## 2) Evaluation Protocol

For each training fraction `f` in {{TRAINING_FRACTIONS}}:

1. Sample `f × |train_set|` examples from the training set (stratified).
2. Train the model on the sampled subset using the same hyperparameters as the full run.
3. Evaluate on the **full** validation set.
4. Record both train-set metric (computed on the sampled subset) and validation-set metric.
5. Repeat steps 1-4 for each seed in the stability list.
6. Report **mean +/- std** across seeds for both train and validation metrics.

**Budget note:** Learning curve runs are supplementary analysis. They do not count against the primary experiment budget but MUST be logged separately.

## 4) Interpretation Guide

Use the following diagnostic framework when interpreting learning curves:

| Pattern | Train Metric | Val Metric | Gap | Diagnosis |
|---------|-------------|------------|-----|-----------|
| **Overfitting** | High | Low | Widens with more data or stays large | Model memorizes; needs regularization or simpler model |
| **Underfitting** | Low | Low | Narrow | Model too simple; needs more capacity or features |
| **Data sufficiency** | High | Plateaus before 100% | Narrows and stabilizes | More data unlikely to help; focus on model/features |
| **Data hungry** | Increasing | Increasing, not plateaued | Narrowing | Model benefits from more data; collect more if possible |

Every algorithm's learning curve MUST receive an interpretation paragraph in FINDINGS.md referencing this table.

---

## 5) Figure Requirements

- **One figure per algorithm** (or per algorithm-dataset pair if multiple datasets).
- **X-axis:** Number of training samples (or fraction). Label: `Training Samples` or `Training Fraction`.
- **Y-axis:** Primary metric value. Label: metric name (e.g., `Balanced Accuracy`).
- **Two lines per figure:**
  - Train metric: **solid** line.
  - Validation metric: **dashed** line.
- **Shaded region:** +/-1 standard deviation across seeds (use `fill_between`).
- **Legend:** `Train` and `Validation` with line style indicators.
- Figures MUST be generated by `gen_learning_curves.py` from JSON data files, not hardcoded.
- JSON source: `outputs/learning_curves/{algorithm}_seed{seed}.json`
