## 1) Hyperparameter Selection

For each algorithm, identify **one** primary complexity-controlling hyperparameter:

| Algorithm | Complexity HP | Low Complexity | High Complexity |
|-----------|--------------|----------------|-----------------|
| Decision Tree | `max_depth` | 1 (stump) | `None` (unlimited) |
| Random Forest | `n_estimators` | 1 | 500+ |
| XGBoost | `n_estimators` | 1 | 500+ |
| SVM | `C` | 0.001 | 1000 |
| Neural Network | `hidden_layer_sizes` | (8,) | (256, 128, 64) |
| kNN | `k` | 1 (high complexity) | N (low complexity) |

**Note:** For kNN, complexity is **inversely** related to `k`. The X-axis should still show `k` values but the interpretation is reversed.

Adapt this table to the algorithms used in your project.

---

## 2) Sweep Range

**Values:** {{HP_VALUES}}

- Minimum **5** distinct values per algorithm.
- Values MUST span from underfitting (low complexity) to overfitting (high complexity).
- Values should be spaced to capture the transition region (e.g., logarithmic spacing for `C`).
- All other hyperparameters are held fixed at their default or best-known values.

Example sweep ranges:

```yaml
decision_tree:
  max_depth: [1, 3, 5, 10, 20, null]
svm:
  C: [0.001, 0.01, 0.1, 1.0, 10.0, 100.0]
knn:
  n_neighbors: [1, 3, 5, 11, 21, 51]
neural_network:
  hidden_layer_sizes: [[8], [32], [64], [128], [256, 128]]
```

---

## 3) Evaluation Protocol

For each HP value `v` in the sweep range:

1. Set the complexity HP to `v`. All other HPs remain fixed.
2. Train the model on the full training set.
3. Evaluate on the full validation set.
4. Record both train-set metric and validation-set metric.
5. Repeat steps 1-4 for each seed in the stability list.
6. Report **mean +/- std** across seeds.

**Budget note:** Complexity sweep runs are supplementary analysis. They do not count against the primary experiment budget but MUST be logged separately.

## 4) Figure Requirements

- **One figure per algorithm** (or per algorithm-dataset pair if multiple datasets).
- **X-axis:** Hyperparameter value. Label: HP name and unit (e.g., `max_depth`, `C (log scale)`).
- **Y-axis:** Primary metric value. Label: metric name (e.g., `Balanced Accuracy`).
- **Two lines per figure:**
  - Train metric: **solid** line.
  - Validation metric: **dashed** line.
- **Shaded region:** +/-1 standard deviation across seeds.
- **Sweet spot annotation:** Mark the HP value where validation metric is maximized with minimal train-val gap. Use a vertical dashed line or marker.
- **Legend:** `Train` and `Validation` with line style indicators.
- Use **log scale** on X-axis when HP values span multiple orders of magnitude (e.g., `C`).
- Figures MUST be generated by `gen_complexity_curves.py` from JSON data files, not hardcoded.
- JSON source: `outputs/complexity_curves/{algorithm}_seed{seed}.json`
